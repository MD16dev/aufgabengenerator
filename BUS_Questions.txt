## Operating System Basics

What lies between applications and hardware and manages resources safely? | Operating System | Compiler | Web browser | Database | The OS is the software layer between apps and hardware; it abstracts the hardware and manages resources safely.
What is the main job of the OS from the user perspective? | Easy abstraction to hardware | Compile programs | Render graphics | Manage networks | From the user view the OS provides an easy abstraction to operate the hardware.
Which OS service is NOT one of the 5 main managements? | Password management | Program management | Memory management | File management | The 5 managements are program, memory, file, network and human interface - not password management.
Which component executes programs and understands an instruction set? | CPU | RAM | Bus | GPU | The CPU fetches, decodes and executes instructions defined by its instruction set.
What is the communication layer linking all hardware components? | Bus | Cache | Register | ALU | The bus is the communication layer that links all hardware components.
Which operation may break the system? | Modify device state | Read a file | Print a document | Open a browser | Modifying device state or accessing restricted memory can break the system, so the OS isolates these.
How many protection domains are usually provided? | 2 | 3 | 4 | 1 | Usually two domains: supervisor mode and user mode.
Which mode has the highest privilege level? | Supervisor mode | User mode | Guest mode | Safe mode | Supervisor (S) mode has the highest privilege; all operations are allowed.
Which mode is used by regular user applications? | User mode | Supervisor mode | Kernel mode | Ring 0 | User (U) mode is the lowest privilege level used by programs outside the kernel.
x86 provides how many processor privilege levels? | 4 | 2 | 3 | 1 | x86 provides 4 rings, with ring 0 being the most privileged.
How do applications in User mode switch to Supervisor mode? | System call | Direct jump | Reboot | Fork | A system call provides the controlled interface to switch from user to supervisor mode.
What does the OS check during a system call? | Permission and valid args | File size | Screen resolution | Battery level | The OS checks whether the program is allowed and whether the passed arguments are valid.

What is a key disadvantage of monolithic kernels? | Limited safety | Slow startup | No drivers | Small size | If one component crashes, the whole monolithic kernel crashes - limited safety.
Microkernels move most services to... | User mode as servers | The BIOS | The GPU | The cloud | Microkernels keep only a minimal set in kernel space; other services run in user mode as servers.
What is a key advantage of microkernels? | Enhanced safety | Faster syscalls | More drivers | Larger size | A crashed server does not crash the whole system, giving enhanced safety.
Hybrid kernels are used by which OS? | Windows NT | Linux | Minix | Unikraft | Windows NT is a hybrid kernel (monolithic base with some user-space servers).
Unikernels embed all software in which privilege level? | Supervisor mode | User mode | Ring 3 | Guest mode | Unikernels run kernel and apps together in supervisor mode.

## Processes

A program is best described as... | A recipe (code) | Running execution | A thread | A file | A program is the recipe (code); execution of it is a process.
Which is NOT a way a process can be created? | Deleting a file | User starts program | Parent creates it | System boot | Processes are created by users, by other processes, or at boot - not by deleting files.
A process with no user interaction is called... | Background process | Foreground process | Daemon thread | Kernel task | Background processes run without user interaction.
Background processes that always stay background are called... | Daemons | Threads | Shells | Drivers | Long-lived background processes are called daemons or services.
In UNIX, the root of the process tree is... | init | kernel | bash | systemd | init is the first process created at boot and is the root of the process tree.
Windows process hierarchy is based on... | Handles to children | A full tree | init | PIDs only | Windows gives parents a handle to their children rather than a full tree.
Which is NOT one of the 3 process states? | Compiling | Ready | Running | Blocked | The three states are Ready, Running and Blocked - not Compiling.
The PCB is used to... | Describe the process | Compile code | Manage disks | Render UI | The Process Control Block describes a process and stores its state.
Which memory region grows toward higher addresses? | Heap | Stack | Text | Data | The heap grows toward higher addresses (e.g. via malloc).
Which memory region grows toward 0x00000000? | Stack | Heap | BSS | Data | The stack grows toward lower addresses, down to 0x00000000.
The Page Table Pointer is one of the... | Process registers | File types | Schedulers | Devices | The page table pointer is one of the four key process registers.
Context switch step: registers are saved into... | The PCB | The disk | The cache | The stack | During a context switch the CPU registers are saved into the process's PCB.
Processes are isolated from each other unless... | Explicitly stated | Always shared | In kernel | On boot | Processes are isolated unless the program explicitly shares memory, files, etc.
Why is the process mechanism considered heavyweight? | Costly creation | Too safe | No memory | Single core | Creating a process and setting up communication is costly, making it heavyweight.

## Threads

What executes the instructions within a process? | Thread | Process | Core | Register | A thread is the entity that executes the instructions.
Threads in the same process share their... | Memory | Stack | Registers | Program counter | Threads of one process share the same address space (memory).
Which data is private to a thread? | Program counter | Address space | Global vars | Open files | Each thread has its own program counter, registers, stack and state.
User threads are typically provided by... | Languages/libraries | The BIOS | The disk | The GPU | User threads are provided by languages (Java, Go) or libraries (OpenMP).
Attaching user threads to OS threads is called... | Thread Mapping Model | Context switch | Fork | Paging | The Thread Mapping Model attaches user threads to kernel threads.
In the 1:1 mapping model, each user thread maps to... | One kernel thread | No kernel thread | Many kernels | The disk | In 1:1 each user thread is attached to exactly one kernel thread.
N:1 mapping provides... | Full scheduling control | Parallelism | OS support | More cores | N:1 gives the runtime full scheduling control but no parallelism.
N:M mapping is used in... | Go, Haskell, Erlang | C, Java | Python, PHP | Assembly | N:M maps N user threads to M kernel threads, used in Go, Haskell, Erlang.
User threads sharing one kernel thread can all block due to... | One blocking op | A fork | A signal | A page fault | If one user thread blocks on a kernel thread, all sharing it block too.
The scheduler allocates which resource? | CPU | Disk | RAM | GPU | The scheduler allocates CPU resources to threads.
CPU-bound applications have... | Few IO waits | Many IO waits | No threads | No memory | CPU-bound programs do long computations with few IO waits.
A non-preemptive scheduler runs a thread until it... | Blocks or yields | Times out | Reboots | Crashes | A non-preemptive scheduler runs a thread until it blocks, terminates or yields.
Batch scheduling targets which workload? | CPU-bound | IO-bound | Real-time | Interactive | Batch scheduling targets CPU-bound, long-running jobs.
Interactive scheduling needs preemption for... | Responsiveness | Security | Compilation | Encryption | Preemption keeps interactive (IO-bound) apps responsive to the user.

## Scheduling Algorithms

FCFS stands for... | First-Come First-Served | Fast Cache File System | Fixed Core Frequency | Free CPU Scheduling | FCFS serves threads in order of arrival, non-preemptively.
Shortest Job First is optimal when... | All jobs known in advance | Jobs arrive randomly | No preemption | Single core | SJF is optimal only when all job runtimes are known in advance.
Shortest Remaining Time is the preemptive version of... | SJF | FCFS | Round-Robin | Lottery | SRT is the preemptive variant of Shortest Job First, sorted by remaining time.
Round-Robin assigns each thread a... | Time quantum | Priority | Stack | File | Round-Robin gives each thread a fixed time quantum before switching.
Priority-based scheduling always picks the... | Highest priority | Oldest thread | Smallest job | First arrival | Priority scheduling always runs the thread with the highest priority.
A fair scheduler gives each thread... | Equal CPU time | Highest priority | Smallest quantum | Most memory | A fair scheduler gives every thread the same amount of CPU time.
A lottery scheduler chooses next thread... | Randomly | By priority only | By age | By size | A lottery scheduler picks the next thread randomly, weighted by tickets.
Hard real time means the deadline... | Must be met | Can be missed | Is ignored | Is optional | In hard real time missing a deadline breaks the system (e.g. car brakes).
Soft real time means deadlines... | Can be missed | Must be met | Are fatal | Do not exist | Soft real time can occasionally miss deadlines without breaking the system.
EDF stands for... | Earliest Deadline First | Efficient Data Fetch | External Device Flag | Easy Deadline First | Earliest Deadline First schedules by the soonest deadline.

## Memory Mapping

Swapping memory is NOT applicable for... | Multi-core systems | Single core | Small RAM | Disks | Swapping in/out of storage is not applicable for multi-core systems.

Static relocation patches addresses by adding the... | Base address | Limit | Offset | Stack | Static relocation patches every address by adding the load base address.
Dynamic relocation uses which two registers? | Base and Limit | PC and SP | IP and GP | TLB and MMU | Dynamic relocation uses a Base register and a Limit register in hardware.
Which technique keeps recently used processes in RAM? | Swapping | Compilation | Encryption | Caching | Swapping keeps recently/frequently used processes in RAM and pushes old ones to storage.
Memory fragmentation can be fixed by... | Compaction | Paging | Forking | Linking | Compaction merges small free chunks into one large block.
Fixed-size address spaces allocate a... | Contiguous chunk | Random bits | Single page | TLB entry | Fixed-size spaces allocate one contiguous chunk of the process size.
When heap and stack touch, the process is... | Out-of-Memory | Crashed | Forked | Swapped | When heap and stack meet, the process is Out-of-Memory.
Bitmaps track free memory with... | One bit per chunk | A list | A tree | A hash | A bitmap uses one bit per memory chunk (0 free, 1 used).
Free lists store elements with state, address and... | Size | Color | Priority | Name | Each free-list element stores state, starting address and size.
Best fit allocation searches for the... | Smallest fitting chunk | First fit | Biggest chunk | Random chunk | Best fit searches for the smallest chunk that fits.
Worst fit always inserts into the... | Biggest chunk | Smallest chunk | First chunk | Last chunk | Worst fit always inserts into the biggest chunk to avoid small leftovers.

## Virtual Memory

LRU evicts the page... | Unused longest | Newest | Biggest | Dirtiest | Least Recently Used evicts the page not used for the longest time.
The Buddy allocator splits chunks by... | Powers of two | Primes | Bytes | Pages | The Buddy allocator repeatedly splits chunks into powers of two.
User-space allocators manage free space via a... | Free list | TLB | PCB | Disk | User-space allocators maintain a free list for returned objects.
